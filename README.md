# ジムトモ

[![gymtomo](https://www.gymtomo.com/images/ogp.png)](https://www.gymtomo.com)

ジムトモは「週に何回ジムに通うか」の目標を設定し、友達と進捗を共有できるWebアプリです💪  
LINE上で完結し、位置情報を使ってジムにチェックインできるため、記録に“嘘がつけない”仕様になっています。

### ✅ 特徴

- LINEで完結する簡単な登録プロセス  
- 位置情報によるジムへのチェックイン機能で記録の信頼性を確保  
- 友人と進捗を共有し、楽しく継続できる仕組み

### 🎨 デザイン

このアプリは、友人が企画・開発、自分がデザイン全般を担当しました。  
現職ではデザインに十分な時間が割ける環境でしたが、本プロジェクトでは2人チームで、MVP志向が強く、限られた工数で最大の成果を目指す必要がありました。

「最低限の情報を整理して形にする」「注力する部分と切り捨てる部分を見極める」など、今までとは異なる制約の中でデザインを進めました。

#### 1. コンセプト理解

まずはアイデアの背景をヒアリングし、コンセプトを明確にした上で情報を整理しました。

<img src="/images/concept.png" alt="情報をまとめたnotion" width="600">

#### 2. ワイヤーフレーム作成

MobbinやUI Pocketを参考に、類似サービスを調査。必要な機能や遷移を洗い出し、ワイヤーフレームを作成しました。

<img src="/images/wire.png" alt="ワイヤーフレーム" width="600">

#### 3. デザイン作成

LINE上で完結する体験に合わせ、UIもLINE寄りに設計しました。  
Material 3 Design Kitをベースに、必要なコンポーネントをLINE風に調整・追加して進行。  
特にオンボーディングは何度も友人と議論しながら作り込みました。

<img src="/images/design.png" alt="オンボーディングのデザイン">

#### 4. ロゴ作成

コンセプトやサービス名からイメージを膨らませ、ラフをPhotoshopでブラッシュアップ。  
1案目で「これだ！」というものができ、工数も最小限に抑えられました。

<img src="/images/logo.png" alt="ロゴ" width="400">

#### 5. クリエイティブ作成

LPやOGPのクリエイティブ制作では、PinterestやLP advanceを参考に試行錯誤しながら作成しました。

<img src="/images/ad.jpg" alt="宣伝用画像" width="600">

#### 6. リリース

完成後は友人や会社の同僚に使ってもらい、リアルな反応を得ることができました☺️

### 📈 成果

リリース後、**2ヶ月でユーザー数が100人を突破！**  
日常の中に溶け込む「サボれない」体験を実現できました。

# エペタム

[![apetamu](https://apetamu.com/images/apetamu_ogp.png)](https://apetamu.com)

エペタムはFPSゲーム『Apex Legends』の試合結果を**自動で集計**するWebアプリです🎮  
APIを活用した自動集計、結果の画像出力やURL共有によって、大会主催者が簡単に運営・発信できる仕組みを提供しています。

リリース後は徐々にユーザー数が増加し、一部機能を課金制にすることで**マネタイズにも成功**💰  
**有名VTuberや公式大会でも採用**され、多くの方にご利用いただいています！

- [渋谷ハル氏の投稿](https://x.com/shibuyahal/status/1825174463471866120?s=46&t=XxQ2hhdFh6Oy7Nd03sfJdA)
- [KinoTrope Gamingによる紹介](https://x.com/kinotropegaming/status/1569974245181992960?s=46&t=XxQ2hhdFh6Oy7Nd03sfJdA)

### ✅ 特徴

- APIを活用した自動集計機能  
- 結果をURLで共有可能  
- 任意の画像上に結果を出力できるビジュアル機能付き

### 🎨 デザインについて

本アプリも友人との個人開発で、自分はデザインとフロントエンドを担当しました。  
初の個人開発ということもあり、試行錯誤しながらの取り組みでした📈

#### 1. コンセプト理解

Apexの大会では、主催者が手作業でリザルトを集計しており、かなりの負担になっていました。  
そこで「誰でも簡単に大会を開ける」体験を目指して、開発者と議論しながら要件と体験を整理しました。

#### 2. デザイン作成

主催者の手間を最小化するため、直感的でシンプルな構成を重視しました。

- タイミング別に必要な操作が明確になるよう導線を設計  
- Xでの大会投稿や参加者の行動を分析し、理想体験をワイヤーに落とし込み  
- レスポンシブデザインでスマホ・PC両対応

<img src="/images/apetamu.png" alt="エペタムのデザイン">

#### 3. 改善サイクル

リリース後も、大会主催者にヒアリングしたり、自分も運営に関わったりしてリアルな課題を収集。  
「ここが使いづらい」「この機能が欲しい」といった声をもとに、UIの改善と機能追加を継続的に行ってきました。

- [Xでの機能実装告知](https://x.com/apetamu_jp/status/1543901147660193792)

### 📈 成果

開発開始から約1年でリリースし、**ユーザー数5,000人を突破！**  
有名VTuber主催の大会や、公式イベントでも利用され、**月間黒字**を達成しました🎉  
「運営の負担が減った」「見せやすくなった」といった声も多く、目指した体験の実現に手応えを感じています💪
